# Final Validation Strategy

Final validation proves that the project is ready for delivery.

Day 2 checked whether Silver data is trustworthy.

Day 5 checks whether final business outputs are ready to present.

You should validate:

- required Gold tables exist
- dashboard views return rows
- important KPIs are reasonable
- fraud feature table has one row per claim
- dashboard views do not expose raw PII
- quarantine tables exist
- README and documentation match the actual project
